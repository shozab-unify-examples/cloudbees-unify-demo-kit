# Candidate Notes

Every workflow in this kit is self-contained and mock/echo-based, so it runs green in your tenant with no extra setup — no clusters, secrets, registries, or integrations. Fork, connect, and click **Run workflow**. Use these as realistic building blocks for a customer-facing CloudBees Unify demo, then tailor the story.

Recommended customization:

- Rename the application to match your selected scenario.
- Change component names to fit your story.
- Add or remove a tool integration from the story.
- Add one manual approval step in the Unify composer if your environment supports it.
- Show how the same pattern works across teams without forcing them to replace GitHub Actions, Jenkins, GitLab, Jira, or ServiceNow.
